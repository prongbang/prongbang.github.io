### CallX คืออะไร

CallX เป็นไลบรารี HTTP client สำหรับภาษา Go ที่ช่วยให้เรา **เรียก API ได้ง่ายขึ้นด้วยโค้ดไม่กี่บรรทัด** โดยด้านในใช้ `fasthttp` และออกแบบ API ให้ใช้งานตรงไปตรงมา เช่น `Get`, `Post`, `Put`, `Patch`, `Delete` และ `Req` สำหรับ request แบบ custom

เหมาะกับงานที่ต้องเรียก REST API บ่อย ๆ และอยากได้ client ที่ตั้งค่าได้ง่าย เช่น base URL, timeout, header, bearer token, form encoded, interceptor หรือ cookies

ติดตั้งได้ด้วยคำสั่ง

{% highlight shell %}
go get github.com/prongbang/callx
{% endhighlight %}

---

### แนวคิดหลัก

* ใช้ `callx.Config` เพื่อกำหนดค่า client เช่น `BaseURL`, `Timeout`, `Name` และค่า performance อื่น ๆ
* ใช้ `callx.New` เพื่อสร้าง client
* ใช้ method สำเร็จรูป เช่น `Get`, `Post`, `Put`, `Patch`, `Delete`
* ใช้ `Req` เมื่อต้องการกำหนด method, header, body หรือ form เอง
* ใช้ interceptor เพื่อเติม header, content type หรือ log request/response
* อ่านผลลัพธ์จาก `callx.Response` ที่มี `Code`, `Data` และ `Cookies`

---

### ตัวอย่างการใช้งาน

#### 1. สร้าง client พร้อม Base URL

{% highlight go %}
package main

import (
  "fmt"

  "github.com/prongbang/callx"
)

func main() {
  c := callx.Config{
    BaseURL: "https://jsonplaceholder.typicode.com",
    Timeout: 60,
  }

  req := callx.New(c)

  data := req.Get("/todos/1")
  fmt.Println(data.Code)
  fmt.Println(string(data.Data))
}
{% endhighlight %}

จากตัวอย่างนี้ `BaseURL` จะถูกนำมาต่อกับ path ที่ส่งเข้าไปใน `Get` ทำให้ไม่ต้องเขียน host ซ้ำทุกครั้ง

---

#### 2. เรียก POST แบบส่ง JSON Body

{% highlight go %}
body := callx.Body{
  "username": "root",
  "password": "pass",
}

data := req.Post("/login", body)
fmt.Println(data.Code)
fmt.Println(string(data.Data))
{% endhighlight %}

ถ้า body ไม่ใช่ form, CallX จะ marshal payload เป็น JSON ก่อนส่ง request ให้

---

#### 3. Custom Request พร้อม Header

ถ้าต้องการกำหนด URL, method, header และ body เอง สามารถใช้ `callx.Custom` คู่กับ `Req` ได้

{% highlight go %}
custom := callx.Custom{
  URL:    "https://httpbin.org/post",
  Method: http.MethodPost,
  Header: callx.Header{
    callx.Authorization: fmt.Sprintf("%s %s", callx.Bearer, "your-token"),
  },
  Body: callx.Body{
    "username": "root",
    "password": "pass",
    "address": []string{
      "087654321",
      "089786756",
    },
  },
}

data := req.Req(custom)
fmt.Println(data.Code)
fmt.Println(string(data.Data))
{% endhighlight %}

วิธีนี้เหมาะกับ request ที่ต้องใช้ header พิเศษ เช่น `Authorization`, `Content-Type` หรือ endpoint ที่ไม่ได้อยู่ภายใต้ `BaseURL`

---

#### 4. ส่งข้อมูลแบบ Form Encoded

สำหรับ API ที่รับ `application/x-www-form-urlencoded` สามารถส่ง form ผ่าน `strings.NewReader` ได้

{% highlight go %}
form := url.Values{}
form.Set("message", "Test")

custom := callx.Custom{
  URL:    "https://httpbin.org/post",
  Method: http.MethodPost,
  Header: callx.Header{
    callx.Authorization: "Bearer XTZ",
    callx.ContentType:   "application/x-www-form-urlencoded",
  },
  Form: strings.NewReader(form.Encode()),
}

data := req.Req(custom)
fmt.Println(data.Code)
fmt.Println(string(data.Data))
{% endhighlight %}

ถ้ากำหนด `Form` ใน `Custom` ตัว CallX จะส่ง body แบบ stream จาก reader แทนการ marshal JSON

### Interceptor

CallX มี interceptor สำหรับจัดการ request/response ก่อนและหลังยิง API โดยตัวที่มีให้ใช้งาน เช่น

* `HeaderInterceptor` สำหรับเติม header กลางให้ทุก request
* `JSONContentTypeInterceptor` สำหรับตั้ง `Content-Type` และ `Accept` เป็น `application/json`
* `LoggerInterceptor` สำหรับ log request และ response

ตัวอย่างการเพิ่ม interceptor

{% highlight go %}
c := callx.Config{
  BaseURL: "https://jsonplaceholder.typicode.com",
  Timeout: 60,
}

req := callx.New(c)

req.AddInterceptor(
  callx.JSONContentTypeInterceptor(),
  callx.HeaderInterceptor(callx.Header{
    callx.Authorization: "Bearer your-token",
  }),
  callx.LoggerInterceptor(512),
)

data := req.Get("/todos/1")
fmt.Println(data.Code)
{% endhighlight %}

ถ้าต้องการ log body ไม่ให้ยาวเกินไป สามารถกำหนดค่า max body log ให้ `LoggerInterceptor` ได้ เช่น `512`

### อ่าน Cookies จาก Response

ถ้าต้องการอ่าน cookies จาก response ให้เปิด `Cookies` ใน config

{% highlight go %}
c := callx.Config{
  BaseURL: "https://example.com",
  Timeout: 60,
  Cookies: true,
}

req := callx.New(c)

data := req.Get("/profile")
fmt.Println(data.Cookies)
{% endhighlight %}

เมื่อเปิด `Cookies: true` ค่า cookies ที่ได้จาก response จะถูกเก็บไว้ใน `Response.Cookies`

---

### Config ที่น่าสนใจ

นอกจาก `BaseURL` และ `Timeout` แล้ว CallX ยังมี config สำหรับปรับ behavior ของ client ได้อีก เช่น

* `Name` สำหรับกำหนดชื่อ client ใน `User-Agent`
* `ReadTimeout` และ `WriteTimeout`
* `MaxIdleConnDuration`
* `MaxConnsPerHost`
* `ReadBufferSize` และ `WriteBufferSize`
* `RetryIf`
* `StreamResponseBody`
* `TLSConfig`
* `InsecureSkipVerify`

โดยทั่วไปเริ่มจาก `BaseURL` และ `Timeout` ก่อนก็พอ จากนั้นค่อยเพิ่ม config อื่นเมื่อมี requirement เฉพาะ เช่น ต้อง tune connection, ใช้ TLS config เอง หรือเปิด stream response

### เหมาะกับงานแบบไหน

* เรียก REST API จาก service ภาษา Go
* ทำ API client wrapper ภายในโปรเจกต์
* เรียก API ที่ต้องใส่ bearer token หรือ custom header
* เรียก endpoint หลายตัวที่ใช้ base URL เดียวกัน
* ต้องการ log request/response ระหว่าง debug
* ต้องการ HTTP client ที่ API ใช้งานง่ายและตั้งค่าได้

---

### ข้อควรรู้

* ถ้ากำหนด `BaseURL` แล้ว path ที่ส่งเข้า `Get`, `Post` หรือ method อื่น ๆ จะถูกนำไปต่อท้าย base URL
* `Response.Data` เป็น `[]byte` ถ้าต้องการใช้งานเป็น struct ให้ unmarshal เองตาม type ที่ต้องการ
* ถ้า request error ตัว response จะคืน `Code` เป็น `http.StatusNotFound` พร้อมข้อความ error ใน `Data`
* `InsecureSkipVerify` ควรใช้เฉพาะกรณีที่จำเป็น เช่น local development หรือระบบทดสอบเท่านั้น
* `LoggerInterceptor` เหมาะกับ debug แต่ควรระวังข้อมูล sensitive ใน production

---

### สรุป

CallX เป็น HTTP client ที่ช่วยลด boilerplate เวลาเรียก API ใน Go ได้ดี โดยยังเปิดให้ปรับแต่งได้หลายส่วน ทั้ง base URL, timeout, custom request, form body, header, interceptor และ cookies

ถ้างานของเราต้องเรียก API หลาย endpoint และอยากได้ client ที่เรียกง่าย อ่านง่าย และต่อยอดเป็น API wrapper ได้ทันที CallX ก็เป็นอีกตัวที่ใช้งานได้สะดวกมาก
